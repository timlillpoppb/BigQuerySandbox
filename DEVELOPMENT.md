# Development Guidelines

## Naming Conventions

- Models: `stg_`, `src_`, `brz_`, `slv_`, `fct_`, `dim_`
- Columns: snake_case
- Tests: `test_<model>_<check>`

## Best Practices

- Use incremental models where possible
- Add contracts for model interfaces
- Test coverage >80%
- Document all models with descriptions

## Checklist

- [ ] Model compiles
- [ ] Tests pass
- [ ] Documentation updated
- [ ] Peer review completed
